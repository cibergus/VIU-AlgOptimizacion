Programacin Dinmica. Viaje por el rio

  • Definicin: Es posible dividir el problema en subproblemas ms pequeos, guardando las soluciones para ser utilizadas ms adelante.
  • Caractersticas que permiten identificar problemas aplicables:
    -Es posible almacenar soluciones de los subproblemas para ser utilizados ms adelante
    -Debe verificar el principio de optimalidad de Bellman: en una secuencia optima de decisiones, toda sub-secuencia tambin es ptima (*)
    -La necesidad de guardar la informacin acerca de las soluciones parciales unido a la recursividad provoca la necesidad de preocuparnos por la complejidad espacial (cuantos recursos de espacio usaremos)

Problema

En un ro hay n embarcaderos y debemos desplazarnos ro abajo desde un embarcadero a otro. Cada embarcadero tiene precios diferentes para ir de un embarcadero a otro situado ms abajo. Para ir del embarcadero i al j, puede ocurrir que sea ms barato hacer un trasbordo por un embarcadero intermedio k. El problema consiste en determinar la combinacin ms barata.

  • Resuelve el problema del descenso por el rio utilizando la tcnica de optimizacin que consideres ms adecuada.

# Embarcaderos y costos conocidos (datos extrados de la imagen)
n_embarcaderos = 7
caminos_costos = [(0, 1, 5), (0, 2, 4), (0, 3, 11), 
                  (1, 3, 2), (1, 4, 3),
                  (2, 3, 1), (2, 4, 5), (2, 5, 6),
                  (3, 4, 3), (3, 5, 4), (3, 6, 11), 
                  (4, 5, 4), (4, 6, 11), 
                  (5, 6, 3)
                 ]

# Inicializacin costos y rutas
costos = np.full((n_embarcaderos, n_embarcaderos), np.inf)
rutas = np.full((n_embarcaderos, n_embarcaderos), None)

# Establecemos costos directos y cero para la misma posicin
for i, j, costo in caminos_costos:
    costos[i][j] = costo
for i in range(n_embarcaderos):
    costos[i][i] = 0

# Relacin de recurrencia y relleno de la tabla con ruta
for k in range(n_embarcaderos):
    for i in range(n_embarcaderos):
        for j in range(n_embarcaderos):
            if costos[i][j] > costos[i][k] + costos[k][j]:
                costos[i][j] = costos[i][k] + costos[k][j]
                rutas[i][j] = k

# Recuperamos y reconstruimos la ruta ptima utilizando la matriz de rutas
def reconstruir_ruta(rutas, origen, destino):
    # Si no hay ruta intermedia
    if rutas[origen][destino] is None:
        return []
    k = rutas[origen][destino]
    return reconstruir_ruta(rutas, origen, k) + [k] + reconstruir_ruta(rutas, k, destino)

# Usamos la matriz de rutas para encontrar el camino ptimo del embarcadero 0 al 6
embarcadero_inicio = 0
embarcadero_final = 6
camino_optimo = [embarcadero_inicio] + reconstruir_ruta(rutas, 0, 6) + [embarcadero_final]
costo_minimo = int(costos[embarcadero_inicio][embarcadero_final])
print(f"El camino ptimo del embarcadero {embarcadero_inicio} al {embarcadero_final} con un costo de {costo_minimo} es: {camino_optimo}")

# Definimos el gradiente de la funcin
def df(X):
    x, y = X
    dfdx = np.cos(1/2 * x**2 - 1/4 * y**2 + 3) * np.cos(2*x + 1 - np.exp(y)) + \
           (x * np.sin(1/2 * x**2 - 1/4 * y**2 + 3) * np.sin(2*x + 1 - np.exp(y)))
    dfdy = (-1/2 * y) * np.cos(1/2 * x**2 - 1/4 * y**2 + 3) * np.cos(2*x + 1 - np.exp(y)) - \
           np.exp(y) * np.sin(1/2 * x**2 - 1/4 * y**2 + 3) * np.sin(2*x + 1 - np.exp(y))
    return np.array([dfdx, dfdy])

# Fijamos la semilla + punto inicial aleatorio + tasa de aprendizaje
np.random.seed(0)
P = np.random.uniform(-2, 2, 2)
puntos = [P.copy()]
TA = 0.01

# Realizamos las iteraciones del descenso del gradiente
for _ in range(1000):
    grad = df(P)
    P -= TA * grad
    puntos.append(P.copy())
puntos = np.array(puntos)
